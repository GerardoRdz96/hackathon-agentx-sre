# Solution Introduction — Incidex

When a payment gateway fails at 2 AM, the on-call SRE engineer faces a critical bottleneck: reading logs, scanning code, forming hypotheses, classifying severity, and routing to the right team — all while the site bleeds revenue. Industry data shows this manual triage process averages 47 minutes before a fix even begins. For e-commerce platforms processing thousands of transactions per minute, every second of downtime translates directly to lost revenue and eroded customer trust.

Incidex solves this with a 5-agent Claude pipeline that performs the entire triage workflow in 6–8 seconds — a 91% reduction. An incident report (text and optional screenshot) enters the system and flows through five specialized agents: a Triage agent classifies severity and component, a Log Analyst and Code Analyst work in parallel to scan 200+ log entries and 9 source files for root cause patterns, a Hypothesis Engine synthesizes findings into ranked hypotheses with confidence scores, and a Router assigns the ticket to the correct engineering team with real email and Telegram notifications. Each agent uses the right Claude model for its task — Haiku for fast classification, Sonnet for deep code reasoning — keeping cost under $0.02 per incident while maintaining production-grade analysis quality.

What sets Incidex apart is that it is not a hackathon prototype — it is built on battle-tested patterns from PA·co, a production multi-agent operating system with 12 agents across 5 departments that has autonomously shipped multiple SaaS products. The architecture includes real integrations (Resend email, Telegram Bot API), 5-layer prompt injection guardrails, full observability (structured logs, per-agent traces, metrics endpoint), and a simulated Medusa.js e-commerce environment that ensures reproducible evaluation. Judges can run the full end-to-end flow with a single `docker compose up --build` — no external services required.
